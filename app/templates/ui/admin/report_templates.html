Help / Guia de uso

Report Templates es un requisito transversal: no pertenece solo a Hoja de Cotejo ni comparte un formato unico. Cada reporte conserva su propio formato actual: Bonafide usa su formato Bonafide, Hoja de Cotejo usa su formato Hoja de Cotejo, Consolidado usa su formato Consolidado, etc.

Esta pantalla sirve para asignar versiones por propuesta y evitar que un cambio nuevo afecte informes culminados.

  • No editar una plantilla base para resolver una propuesta nueva. Crear una version nueva.
  • No romper informes culminados: cada propuesta puede quedar amarrada a una version especifica.
  • Reportes sin asignacion: usan su propio formato base por defecto, no el formato de otro reporte.
  • Propuestas actuales: deben quedar asociadas al formato base actual de cada reporte cuando se requiera congelar/versionar.
  • Reportes futuros: deben agregarse a este catalogo y resolver su configuracion por report_key desde el inicio.
  • Cambios permitidos por version: header, logos/fotos, footer, columnas, etiquetas, orden y reglas visuales.
Nota tecnica: algunas pantallas ya pueden consultar la plantilla; otras quedan registradas aqui como requisito operativo para cuando se migren a plantillas versionadas.
